Chaos Monkeys: Software Engineering at Scale

This project was part of my undergraduate thesis, focusing on researching & applying principles of chaos engineering on a web cloud server.

As Netflix puts it, imagine “unleashing a wild monkey with a weapon in your data center (or cloud region) to randomly shoot down instances and chew through cables”. If they could handle such random anomalies during the workday without customer impact, they could be much more confident that their procedures would work.

A vast array of challenges arise when attempting to build a software service at scale. Users may access the service from all over the world and should see small latencies no matter their location. Data centers or web endpoints may go down unexpectedly, but traffic must flow. High traffic from individual users should not impact other users. And when something goes wrong (and it will), monitoring and alerting will be in place to be able to detect the problem and recover as quickly as possible.

As part of my thesis, my team and I focused on 5 practices of chaos engineering: External Monitoring via status pages, Rate Limiting, Fault Injection, Database Sharding, and Load Testing.

This project was completed during Fall 2024 - Spring 2025 for my Undergraduate Thesis at Carleton College.

Scheme Interpreter

I built a fully functional Scheme interpreter in C that processes and evaluates Scheme code through three main stages: tokenization, parsing, and evaluation. The interpreter begins by tokenizing the input string, breaking it into a sequence of typed tokens such as integers, symbols, and parentheses. These tokens are then parsed into a tree that represents the grammatical structure of the code based on Scheme’s syntax. Finally, the tree is evaluated according to the rules of Scheme, supporting constructs like variables, conditionals, recursion, lambda expressions, and list operations. To manage memory, I implemented a basic memory allocation tracking system that records each allocation made during execution. At the end of the program's run, the interpreter automatically frees all allocated memory in a single cleanup step.

This project was completed during in the Fall of 2024 for CS:251 - Programming Langauges Design & Implementation at Carleton College.

Movie Recommender

Developed a movie recommendation system using neural networks built with Keras and Scikit-learn to predict user preferences based on demographic data and movie genres. The project included both a binary classification model to predict whether a user would like a movie and a multiclass model to estimate exact ratings. Despite using a limited subset of data (~2,000 rows), the binary model outperformed baseline benchmarks after hyperparameter tuning, while the multiclass model showed limited improvement. Performance was evaluated using cross-validation, and high variance was attributed to data constraints.

This project was completed during in the Winter of 2024 for CS:320 - Machine Learning at Carleton College.

Clone Wordle

I developed an interactive website that replicates the New York Times Wordle game using HTML, CSS, and JavaScript to create a responsive and engaging front-end experience. The application includes full game logic, dynamic UI updates, and visual feedback for user guesses. I implemented a Flask server in Python to manage game state and user statistics, which were stored and retrieved using MongoDB to support flexible and scalable data handling. Docker was used to containerize the application, making it easy to deploy and test the full stack in consistent environments.

Language Communities in New York City

I developed an interactive website that allows users to explore over 1,200 cultural communities across New York City, using HTML, CSS, JavaScript, and Bootstrap to create a responsive and user-friendly interface. The front end allows users to search by language, neighborhood, language group, country, and more, displaying a dynamic table of relevant language communities based on their selections. On the back end, I implemented a Flask server in Python to handle client requests and connect to a PostgreSQL database, where community data is stored and queried using SQL. Bootstrap was used to streamline the layout and styling process, while PostgreSQL enabled efficient storage and retrieval of large datasets with complex relationships between communities and locations.
